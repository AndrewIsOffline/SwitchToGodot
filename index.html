Don't wait for Godot, switch today!

This page is meant for people who are ditching Unity after their recent fees, but are finding the transition to Godot to be daunting. This page lays out the big differences between the two engines in a way that is meant to give former Unity developers a head start into the Godot environment.

Interface

Unity Godot
Layout Unity has a window-based, customizable layout, with the main game window sitting in the middle and various movable windows sitting to the sides and below. Godot has a similar, window-based, customizable layout, though with slightly different defaults. Do note that the "run" button is off to the side, with the main window "mode" sitting in the middle.
Running Clicking the "run" button starts the current scene. The scene runs in the game window and the side windows are updated as in-game events happen. Clicking the "run" button builds the game and starts the current scene. The scene runs in a separate window, and side windows are updated as in-game events happen, though going to the "scene" window and selecting "local" will let you view how the scene originally was.
Updating files After updating files externally, clicking on the Unity window will load files and compile scripts. Any script errors will be shown now. After updating files externally, clicking on the Godot window will load files. Scripts are compiled when the game is run.
Runtime window features Unity can be switched back to the editor window during runtime and navigate the scene while it is running. Godot can add debugging effects that are visible during runtime, like highlighting collision points, through the "Debug" menu.
UI Elements UI elements must be attached to a canvas object, and appear within the editor. UI elements are freestanding, and appear in the 2D editor, even in 3D games.

Structure

Unity Godot
Gameplay Elements Unity has "GameObjects", which by themselves only contain coordinates and gain behaviors through "Components", which perform various actions like drawing meshes or constraining physics bodies. Godot has "Nodes", which each perform a single action like drawing meshes or constraining physics objects. This makes for a messier heirarchy, but since each node has its own transform, more versatility is allowed in object placement. Scripts, however, are attached to nodes.
Levels and Prefabs Unity has "Levels", which contain game objects as well as environment settings and are meant to replace each other as separate "levels" of a game. Unity also has "prefabs", which also contain game objects and are meant to be loaded as pieces of a game, like obstacles or pickups. Godot has "Scenes", which contain game objects. Besides including a node for environment data, nothing separates levels from prefabs.
Meshes as colliders Meshes can be used as colliders. Meshes must be converted to colliders. This is easily done by creating a MeshInstance3D, opening the "Mesh" menu (above the main window), and selecting one of the "Create Collision Sibling" options.
Object labeling Unity uses "tags", which act as an attribute of an object. Each object can only have one tag at a time. Godot uses "groups", which act as collections of objects. Each object can belong to multiple groups.
Collision Masking Objects can be in one of 32 layers. A boolean matrix dictates which layers interact with each other. Each object has a bitfield representing the layers that it is a part of and a mask representing the layers that it collides with.

Coding

Unity Godot
Languages Unity supports C#. Godot supports C#, C++, and GDscript, a language created specifically for the engine. Within the documentation, GDScript has the most support, but C# is a close second. Do note that Godot has a separate version for C# support, though it is cross-compatable with the main version. Also note that while C# has most of the same features that GDscript does, they are written differently: GDscript uses snake_case, while C# uses PascalCase.
Shading Languages Unity uses HLSL. It also has "ShaderLab", a node-based wrapper for shaders. Godot uses a GLSL-like language.
IDE Unity relies on external IDEs Godot has an internal IDE, accessable through the "script" button at the top of the editor, though external IDEs also work. The IDE has some integration features with the rest of Godot. For example, dragging an object from the Scene or FileSystem windows into the code inserts the object's location.
Setting variables in-editor Declaring a variable as public in a script adds a setting to the Unity editor to set an intial value. Adding a metadata value in the Godot editor adds it to a map that can be accessed in scripts through GetMeta.
